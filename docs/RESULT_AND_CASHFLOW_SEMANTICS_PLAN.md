# Result And Cashflow Semantics Plan

## Problem

Сейчас в нескольких местах UI смешивает две разные метрики:

- `Операционный результат` = `Доход - Расход`
- `Денежный поток` = `Операционный результат + денежные события долгов + FX cashflow`

Из-за этого:

- пользователь видит большой минус и воспринимает его как `я столько потратил`
- календарь и часть аналитики говорят языком `Доход / Расход`, хотя по факту уже показывают `Приток / Отток`
- дашборд и валютный портфель визуально конкурируют за роль “главного результата”

## Decision

Принять единый паттерн по продукту:

- `Операционный результат` всегда означает только `income - expense`
- `Денежный поток` всегда означает `operations + debts + fx cash side`
- `Валютный портфель` показывает состояние портфеля и оценку, но не маскируется под результат периода

## Scope

### Dashboard

- `КПИ периода`:
  - `Доход`
  - `Расход`
  - `Операционный результат`
  - `Денежный поток`
  - `Операции`
- secondary chips:
  - `Средний расход/день`
  - `Самый затратный день`
  - `Долги в потоке`
  - `Валюта в потоке`

### Analytics

- `Структура / Highlights`: те же 5 KPI
- `Тренды`:
  - график продолжает рисовать unified line, но легенда и tooltip называют ее `Денежный поток`
  - KPI под графиком разделяют `Операционный результат` и `Денежный поток`
- `Календарь`:
  - `Доход / Расход` заменить на `Приток / Отток`, потому что туда уже включены debt/fx inflow/outflow
  - финальную колонку и totals назвать `Денежный поток`
  - убрать смешивание с `результатом валютного портфеля`

### Operations

- обычный режим `Операции`:
  - summary card = `Операционный результат по выборке`
- режим `Денежный поток`:
  - summary card = `Денежный поток по выборке`

### Currency Portfolio

- дашбордовый `Валютный портфель` сделать обзорным:
  - убрать тяжелый KPI-grid
  - оставить карточку оценки портфеля
  - оставить карточки по валютам: `количество`, `курс`, `≈ в базовой валюте`
  - компактный secondary summary для `вложено / нереализованный / реализованный / итог`

## Guardrails

- не менять backend-формулы cashflow без причины
- не подмешивать `currency_total_result_value` в `Денежный поток`
- не использовать слово `Профицит / Дефицит` там, где по факту показывается unified cashflow без явного контекста

## Verification

- сверить dashboard summary, analytics highlights, trend, calendar и operations summary на одинаковом периоде
- убедиться, что labels и tooltip не вводят в заблуждение
- проверить, что валютный портфель не повторяет KPI периода
