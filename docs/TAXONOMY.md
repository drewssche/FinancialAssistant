# Taxonomy

## Domain Taxonomy
- `Operation`: единица учета (доход/расход).
- `Kind`: верхний тип операции (`income`, `expense`).
- `CategoryGroup`: общая категория внутри `kind` (например, `Еда`, `Базовые платежи`).
- `Category` / `Subcategory`: детальная категория внутри `CategoryGroup` (`ЗП`, `Коммуналка` и т.п.).
- `Account`: источник/контур денег (`card`, `cash`, `wallet`, `currency`).
- `Period`: окно анализа (`day`, `week`, позже `month`).
- `Summary`: агрегированный отчет по периоду.

## UI Taxonomy
- `Core Action`: главное действие экрана (например, `+ Добавить операцию`).
- `Primary Action`: подтверждение в рамках потока (`Сохранить`).
- `Secondary Action`: утилитарное действие (фильтр, переключатель периода).
- `Ghost Action`: нейтральные действия (`Отмена`, `Закрыть`).
- `Danger Action`: разрушительные действия (`Удалить`).
- `Data Table`: компактное представление списков и агрегатов.
- `KPI Card`: блок с ключевым показателем.
- `Modal`: контекстный слой ввода/редактирования.

## Logic Taxonomy
- `Create Operation`: валидация и сохранение операции.
- `Summarize`: расчет агрегатов по периоду.
- `Filter`: ограничение выборки по дате/типу/категории.
- `Validate`: правила данных (сумма > 0, корректная дата, допустимая подкатегория).
- `Format`: представление сумм/дат для UI.

## Naming Convention
- Backend сущности: `snake_case`.
- API пути: `/api/<resource>`.
- Frontend data-атрибуты: `data-*`.
- UI классы: семантические (`primary-btn`, `danger-btn`, `kpi-card`), не стилистические.
