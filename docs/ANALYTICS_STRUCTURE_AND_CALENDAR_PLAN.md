# План: структура аналитики и календарь

## Проблемы

- В `Аналитика -> Структура` режим `Все` визуально смешивает доходы и расходы в одном donut как одну общую сумму.
- Такая подача математически допустима, но смыслово неочевидна: пользователь ожидает видеть либо раздельную структуру, либо явное разделение типов потока.
- В `Аналитика -> Календарь` ячейки месяца раздуваются из-за inline-маркеров `Долг ...` и `FX ...`.
- Из-за этого сетка теряет аккуратность, хотя debt/fx-деталь полезнее как вспомогательная расшифровка, а не как основной контент клетки.

## Решение

### 1. Структура

- Не смешивать debt/fx в donut структуры категорий.
- Оставить структуру категорий только для категоризируемых операций.
- Для режима `Все`:
  - в центре donut показывать не просто `Итог периода`, а явное `Доходы + Расходы`;
  - под суммой показывать компактную split-мету: `Доход X · Расход Y`;
  - в списке breakdown для `Все` сохранять текущие строки, но усилить маркировку типа (`Доход` / `Расход`), чтобы было очевидно, почему рядом оказались категории разных знаков.
- Debt/FX не включать в donut структуры, потому что это не category taxonomy и их смешивание с категориями делает график менее читаемым.

### 2. Календарь

- Оставить в клетке только:
  - дату
  - приток
  - отток
  - число событий
- Убрать inline-строку `Долг ... · FX ...` из тела ячейки.
- Перенести debt/fx-деталь в tooltip / `title` кнопки дня и в secondary summary above-grid.
- В weekly/monthly/yearly totals сохранять `Денежный поток` и расшифровку debt/fx отдельными chips.

### 3. Семантика KPI

- Для календаря итоговая колонка и totals продолжают считать unified `Денежный поток`.
- При этом:
  - `Приток` = операции + положительный debt cashflow + положительный FX cashflow
  - `Отток` = операции + отрицательный debt cashflow + отрицательный FX cashflow
- Проверить, что day/week/month/year используют одну и ту же `resolveCashflowTotals` семантику без отдельных визуальных отклонений.

  - год
