# Dashboard KPI Period Plan

## Context

`КПИ периода` на дашборде сейчас считает:

- `Доход`
- `Расход`
- `Баланс = income_total - expense_total`
- `Операции`

При этом выше на дашборде уже есть отдельный блок `Валютный портфель`, который отвечает за:

- текущую оценку валютных позиций
- вложено
- прибыль / убыток
- позиции по валютам

Из-за этого слово `Баланс` в `КПИ периода` читается слишком буквально, хотя по смыслу это именно результат потока за период, а не точный остаток денег на счетах.

## Decision

Переопределить смысл третьей KPI-карточки в блоке `КПИ периода`:

- если значение `> 0` -> `Профицит`
- если значение `< 0` -> `Дефицит`
- если значение `= 0` -> `Ноль`

Сумма при этом остаётся текущим `income_total - expense_total`.

## UI Changes

### Primary KPI row

Оставить:

- `Доход`
- `Расход`
- `Профицит / Дефицит / Ноль`
- `Операции за период`

Для третьей карточки:

- динамический заголовок
- динамический tone:
  - positive -> green
  - negative -> red
  - neutral -> neutral

### Secondary chips row

Убрать chip, который дублирует новую третью KPI-карточку:

- было: `Профицит/Дефицит: ...`

Оставить:

- `Средний расход/день`
- `Самый затратный день`

## Product Semantics

Блок `КПИ периода` трактуется как:

- финансовый результат потока за выбранный период
- без обещания точного остатка по счетам

Блок `Валютный портфель` трактуется как:

- текущее состояние и оценка валютных активов

Такое разделение убирает дублирование смыслов:

- сверху: `что есть сейчас в валюте`
- ниже: `какой результат по денежному потоку за период`

## Next Step

Добавить дополнительный secondary indicator:

- `С учетом валютных сделок`

Но только как отдельный secondary indicator по cashflow FX-сделок, без смешивания с текущей переоценкой валютного портфеля по курсу НБРБ.

Текущий контракт для этого indicator:

- считается по фактическим курсам самих FX-сделок
- `buy` уменьшает cashflow базовой валюты
- `sell` увеличивает cashflow базовой валюты
- в основной KPI-ряд не подмешивается
- в `Валютный портфель` не дублируется, потому что там остаётся только состояние и переоценка актива
