# Currency PnL Timeline Plan

## Problem

Текущий раздел `Валюта` показывает в основном статическую оценку открытых позиций:

- `Текущая оценка`
- `Вложено`
- `Прибыль / Убыток`

Но основная цифра результата сейчас читается как итог по всей валютной истории, хотя фактически она ближе к:

- нереализованному результату по тому, что осталось в открытых позициях;
- по текущему курсу на сегодня.

Из-за этого пользователь может видеть расхождение с собственным ощущением реального финансового результата:

- часть убытка или прибыли уже могла реализоваться на продажах;
- текущий курс НБРБ не равен фактическому банковскому курсу;
- без таймлайна не видно, как результат менялся по истории сделок.

## Goal

Разделить валютный результат на понятные слои и добавить исторический таймлайн результата по валютной истории.

## Product Contract

### Summary KPI

В разделе `Валюта` верхний summary должен показывать:

- `Текущая оценка`
- `Вложено в открытые позиции`
- `Нереализованный результат`
- `Реализованный результат`
- `Итоговый результат`
- `Открытых позиций`

Где:

- `Нереализованный результат = current_value - book_value` по открытым позициям
- `Реализованный результат = прибыль/убыток, зафиксированные на продажах`
- `Итоговый результат = realized + unrealized`

### Timeline

Добавить график в regular section `Валюта`, не только в аналитике.

Первый этап:

- график `Итоговый результат`
- расчёт по истории сделок и историческим курсам
- если выбран фильтр по валюте, график строится по одной валюте
- если выбран `Все`, график агрегируется по всем валютам

На каждую дату:

- учитываются все сделки до этой даты
- учитывается realized pnl по продажам
- остаток позиции оценивается по последнему доступному курсу на эту дату
- итог = realized + unrealized

## UX Rules

- график в `Валюта` должен объяснять динамику результата, а не дублировать просто историю курса
- для пустой истории показывать честное empty state
- KPI `Итоговый результат` и график должны использовать одинаковую логику расчёта
- `Валютный портфель` на дашборде остаётся отдельным блоком состояния, без попытки заменять этот раздел

## Scope For First Pass

1. Backend:
- добавить summary totals: realized / unrealized / total
- добавить endpoint истории результативности валютного портфеля

2. Frontend section `Валюта`:
- обновить KPI-ряд
- показать realized / unrealized / total в карточках валют
- добавить график `Итоговый результат`

3. Tests:
- API тесты на realized/unrealized/total
- API тест истории pnl

## Later

Отдельным следующим этапом можно добавить переключатель графика:

- `Итоговый результат`
- `Нереализованный результат`
- `Курс`

Но первым проходом достаточно именно `Итогового результата`, потому что он закрывает основную пользовательскую боль.
