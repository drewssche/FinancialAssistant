# ADR-002: UI Action Hierarchy

## Status
Accepted

## Context
Интерфейс финансового учета быстро перегружается действиями. Нужна четкая иерархия кнопок.

## Decision
Вводится фиксированная иерархия действий:
- Core Primary (`+ Добавить операцию`) - один на страницу.
- Primary (`Сохранить`) - подтверждение внутри флоу.
- Secondary (`День/Неделя`, фильтры).
- Ghost (`Отмена/Закрыть`).
- Danger (`Удалить`) с обязательным подтверждением.

## Consequences
### Positive
- Быстрее считывается приоритет действий.
- Ниже риск UX-конфликтов и случайных действий.

### Trade-offs
- Ограничение творческой свободы в локальных экранах.
- Нужно строго соблюдать правила при новых фичах.
